# 自适应本地处理引擎设计

## 背景与目标

当前应用已经具备本地 Faster Whisper、外部 Python worker、CPU/CUDA 选择、长音频分块、任务取消、人工审核和桌面打包能力，但本地执行链路仍存在几类确定的性能与实用性问题：

- 超过 30 分钟的本地音频按块转写时，每个块都会重新创建 Faster Whisper 模型。
- 外部 worker 路径会为每个块重新启动 Python 进程并重新加载模型。
- 音频先编码为 44.1kHz 双声道 MP3，长音频随后再次编码为 16kHz 单声道分块。
- 本地重任务使用全局串行锁，让部分不冲突的轻任务和远程任务一起等待。
- 取消只能在阶段或块之间生效，正在运行的 FFmpeg、worker 或单块推理不能及时停止。
- 已完成块缺少稳定检查点，中途失败可能重复大量计算。
- 本地转写进度较粗，不能稳定显示实时倍率和预计剩余时间。
- 初稿帧、审核候选帧和候选哈希会启动大量独立 FFmpeg 进程，并存在重复截图。
- 原视频副本、MP3、转写分块、候选帧、版本帧和 ZIP 会造成重复磁盘占用。

目标是将固定参数、粗粒度串行处理升级为自适应本地处理引擎，同时适用于普通 CPU 电脑和 NVIDIA GPU 电脑，并覆盖几分钟到 8 小时的视频。默认策略为“均衡”，同时提供“极速”和“高精度”。普通用户不需要理解 beam size、compute type、线程数或块大小。

## 非目标

- 不增加云端账号、云同步或多人协作。
- 不做多机、分布式或服务器级多 GPU 调度。
- 不构建完整视频剪辑时间线。
- 不用新的远程服务替代本地 Faster Whisper。
- 不为追求吞吐而允许多个模型无约束争抢显存或内存。

## 产品模式

### 极速

- 优先缩短首次结果时间。
- 使用较轻的解码参数。
- 减少候选搜索和二次校验。
- 允许之后对局部字幕高精度重转。

### 均衡

- 默认模式。
- 根据硬件自动选择 CPU/CUDA、计算精度、线程数和块大小。
- 对失败或低质量块局部重试，不整段重复处理。

### 高精度

- 使用更严格的解码配置。
- 允许更高 CPU、内存和显存占用。
- 为正式字幕、课程和术语密集内容保留二次识别能力。

性能模式是用户概念；底层执行参数由执行计划解析器决定并持久化，保证结果可解释、可恢复。

## 总体架构

处理引擎拆分为七个单元：

1. **硬件画像器**：检测 CPU、内存、CUDA、显存和运行环境。
2. **执行计划解析器**：结合硬件、时长、模式和模型生成不可变执行计划。
3. **音频准备器**：一次性生成用户音频产物和 ASR 工作音频，避免重复转码。
4. **本地 ASR 会话**：管理模型生命周期，在任务内复用模型。
5. **分块检查点存储**：保存块输入指纹、参数、状态和结果，支持恢复与局部重试。
6. **资源调度器**：按 CPU、GPU、FFmpeg、远程 I/O 和轻任务协调资源。
7. **产物与缓存管理器**：管理临时文件、帧复用、按需 ZIP、保留策略和空间预估。

## 执行计划

每个任务生成 `TranscriptionExecutionPlan`，至少包含：

- `performance_mode`
- `device`
- `compute_type`
- `model_identifier`
- `cpu_threads`
- `num_workers`
- `beam_size`
- `best_of`
- `vad_enabled` 与 `vad_parameters`
- `chunk_seconds` 与 `chunk_overlap_seconds`
- `checkpoint_enabled` 与 `resume_enabled`
- `audio_work_format`、`audio_sample_rate`、`audio_channels`

计划写入任务工作目录。恢复任务时，影响转写结果的字段未变化则复用已有块；模型、语言、音频指纹或解码参数变化时，只使不兼容缓存失效。

### 短视频：30 分钟以内

- 默认不分块。
- 优先减少模型启动和磁盘写入。
- 已缓存兼容模型时立即复用。
- 仍写入单块检查点，统一恢复逻辑。

### 中长视频：30 分钟至 2 小时

- 使用约 10 至 15 分钟的块。
- 相邻块保留少量重叠。
- 每块完成后立即持久化。
- 块失败时独立重试。
- 合并时按时间和文本相似度消除重叠重复。

### 超长视频：2 至 8 小时

- 强制启用检查点和恢复。
- 根据内存、设备和模式动态选择块大小。
- 已完成块在应用重启后直接复用。
- 取消时保留成功块。
- 单块连续失败后暂停并显示可重试范围，而不是废弃整个任务。

## 音频准备

音频区分用户产物 `audio.mp3` 和 ASR 工作输入。设计要求：

- 不再把 44.1kHz 双声道 MP3 作为本地 ASR 的唯一中间输入。
- 本地转写路径直接从源视频生成 ASR 输入或分块，不允许再从 `audio.mp3` 二次生成 ASR 分块；实现优先使用一次 FFmpeg 源文件读取同时生成两类输出。
- 长视频直接从源视频生成 ASR 分块，避免“完整 MP3 → 再有损切块”。
- ASR 工作文件位于明确的工作目录，不作为普通下载产物暴露。
- 定稿后可选择保留检查点或清理工作音频。

均衡模式默认使用 16kHz 单声道 FLAC。除非基准测试证明有明确收益，不默认使用会显著放大 8 小时任务磁盘占用的 PCM WAV。

## 模型生命周期

### 内置 Faster Whisper

- 模型缓存键由模型路径、设备和计算精度组成。
- 一个任务内只创建一次兼容模型。
- 相邻任务可以在安全边界内复用相同模型。
- 缓存一次只允许一个活动推理调用，除非基准证明并行安全且有收益。
- 显存不足或加载失败时按明确规则回退，并记录原因。
- 空闲超时、设置变化或显式释放时卸载模型。

### 外部 Python worker

worker 从“一次调用处理一个文件”升级为单任务会话协议：

- 父进程启动 worker，worker 加载一次模型。
- 父进程逐块发送请求，worker 逐块返回结果和进度事件。
- 父进程每块成功后写检查点。
- 取消时终止 worker，已写检查点继续保留。

首版会话限定为单任务生命周期。跨任务常驻只有在环境切换和退出清理验证后再启用。

## 检查点与恢复

任务目录增加：

```text
work/
  asr/
    plan.json
    manifest.json
    chunks/
      chunk_000.flac
    results/
      chunk_000.json
```

`manifest.json` 记录源文件指纹、计划指纹、块时间范围、块指纹、状态、尝试次数、错误摘要和结果路径。

恢复规则：

- 源文件和计划兼容时复用成功块。
- 单块文件变化只使该块失效。
- 模型、语言或关键解码参数变化使转写结果失效，但可以复用音频块。
- 只修改笔记参数或关键帧设置时，ASR 检查点保持有效。
- 取消任务时，running 块恢复为 pending，succeeded 块不变。

最终 `transcript.json` 由成功块确定性合并，并通过临时文件原子替换。

## 进度与预计时间

执行单元报告：音频准备进度、当前块/总块数、当前块和总已处理音频时长、实时倍率、预计剩余时间、设备/模型/精度和缓存命中情况。

内置 Faster Whisper 在消费 segment 迭代器时根据 segment end 更新进度；外部 worker 使用 JSON Lines 事件。前端继续通过现有任务轮询读取，不增加高频独立轮询。样本不足时隐藏预计时间。

## 取消与重试

- FFmpeg 和外部 worker 使用可跟踪的子进程句柄，取消时终止对应进程树。
- 内置推理至少在 segment 和块边界检查取消。
- 取消不删除成功检查点。
- 用户可以继续任务或从头转写。
- 单块失败自动重试；超过上限后进入可恢复失败状态。
- 任何设备或精度回退都必须展示并记录。

## 资源调度

资源锁按阶段而不是整个任务持有：

- 本地 GPU ASR 默认并发 1。
- 本地 CPU ASR 默认并发 1，只有基准证明有收益时才有限并行。
- FFmpeg 重编码限制并发，避免与 CPU ASR 同时占满核心。
- 远程笔记请求不持有本地 ASR 锁。
- Markdown、字幕、元数据、历史读取和下载不进入重任务队列。
- 关键帧受 FFmpeg 资源限制，但不阻塞轻量读操作。

桌面 UI、取消请求和历史读取必须始终保持响应。

## 关键帧优化

- 初稿帧和审核候选帧共享统一候选来源，避免同一时间点重复截图。
- 候选先生成预览尺寸；定稿时复制或按需抽取高质量帧。
- 候选 JPEG 与感知哈希使用同一解码链路或进程内图像读取；不得为每张候选额外启动第二个 FFmpeg 进程计算哈希。
- 多时间点优先批量抽帧，减少 FFmpeg 进程启动。
- 候选索引保存时间点、源视频指纹和参数，兼容时复用。
- 重新生成文字但章节时间未变化时，不默认重抽全部候选。

## 磁盘与产物策略

- 任务开始前估算源视频、MP3、工作音频、候选帧和 ZIP 空间。
- 临时工作目录与正式产物分离。
- ZIP 改为定稿或下载时按需构建，版本切换不无条件重建。
- 同卷文件系统上的笔记版本优先使用硬链接复用相同帧；硬链接不可用时回退复制。
- 定稿后提供“保留恢复数据”和“仅保留最终结果”。
- 展示单任务和全部缓存大小。
- 清理不得删除活动任务或版本仍引用的文件。

## 兼容与迁移

- 旧任务没有 `work/asr` 时仍能载入。
- `audio.mp3`、`transcript.json`、字幕和 ZIP 文件名保持稳定。
- 新字段有默认值，旧元数据不会导致读取失败。
- worker 会话不可用时可回退旧 worker，但必须提示性能降级。
- CPU/CUDA 自动选择失败时，保留用户显式选择优先权。

## 错误处理

- 显存不足：建议更省显存精度、较小模型或 CPU。
- 磁盘不足：显示预计缺口并允许清理检查点。
- 单块失败：展示时间范围并只重试该块。
- worker 异常退出：保留已完成结果并可重启继续。
- 音频块损坏：只重建对应块。
- 计划不兼容：说明哪些配置变化导致缓存失效。
- 取消：区分“已取消且可继续”和“已清除”。

## 验证与测试

单元和集成测试必须覆盖：

- 长音频一个任务内只创建一次内置模型。
- 外部会话一个任务内只加载一次模型。
- 三种模式在 CPU/CUDA 画像下生成确定计划。
- 计划指纹变化正确使缓存失效。
- 成功块恢复时不重复转写。
- 块边界重叠合并不重复字幕。
- 取消保留成功块并重置运行块。
- 只修改笔记配置不使 ASR 缓存失效。
- ZIP 按需构建且不包含工作文件。
- 关键帧兼容缓存能够复用。
- 旧任务目录继续可用。

建立可重复基准，记录模型加载次数、Python/FFmpeg 启动次数、音频准备耗时、实时倍率、峰值内存/显存、磁盘占用、取消响应和恢复后的重复计算量。

首要验收指标：

- 一个长任务内模型加载次数从按块降为 1。
- 已成功块恢复后的重复转写次数为 0。
- 长视频不再经过完整 MP3 后再次有损分块。
- 取消不需要等待整个任务结束。
- 初稿和审核不再重复抽取同一批关键帧。

## 分阶段交付

### 阶段一：确定性高收益

- 执行计划和性能模式模型。
- 内置模型单任务复用。
- 外部 worker 单任务会话。
- 统一 ASR 音频准备。
- 块级检查点、恢复和真实进度。

### 阶段二：资源与恢复体验

- 可中止 FFmpeg/worker。
- 自动硬件画像和参数选择。
- 阶段级资源锁。
- 磁盘预估和清理策略。

### 阶段三：帧与全局缓存

- 候选帧统一来源和批量抽帧。
- 帧缓存复用。
- 按需 ZIP。
- 兼容的跨任务模型缓存与可控释放。

每个阶段都必须保持现有任务、字幕确认、人工审核、版本和桌面打包流程可用。
